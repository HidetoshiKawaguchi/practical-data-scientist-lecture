## 主なファイルの説明
各ファイル、フォルダの概要は以下の通りです。

- `level_6.py`: 予測モデルの構築と評価用のエントリーポイントとなるコード（実験用コード）

## 得られた知見
### EDAの結論
簡単なEDAを実施することにより得た知見を示す。

- 本データセットには欠損値が存在せず、初期の分析を進めるうえで非常に扱いやすいことが確認された。ただし、free sulfur dioxide には一部やや大きな値が見られ、計測ミスや特殊なサンプルが含まれている可能性がある。詳細な調査は必要だが、分析の初期段階では一旦これらのサンプルを外して進めるのも選択肢となるだろう。

- 目的変数である quality は 5 および 6 の値に集中しており、品質の低い 3 や品質の高い 8 はサンプル数が少ない。このため、回帰モデルで予測する際には分布の偏りに配慮した工夫が必要である。また、極端なサンプルに焦点を当てて継続的に探索的データ分析を行うことで、「良いワイン」「悪いワイン」の特徴をより鮮明に抽出できる可能性がある。

- 相関分析の結果、quality と比較的強い関連を示したのは以下の変数であった。alcohol は正の相関が強く、アルコール度数が高いワインほど品質が高く評価される傾向がある。一方、volatile acidity は負の相関が強く、揮発酸が多いワインは品質が下がりやすい。また、sulphates や citric acid も弱めながら正の相関を示した。これらの変数は予測モデルを開発する際に、特徴量選択の候補として活用できると考えられる。

- 各カラムの説明を以下の表に示す。変数名から、生成AIや検索により調査した一般的な事項を説明として記載している。より詳細には、お客様の専門家に正しいか否かは確認する必要がある。

| 変数名 (英語)          | 日本語名       | 説明 |
|----------------------|----------------|------|
| fixed acidity        | 固定酸度       | 主に酒石酸など揮発しない酸。ワインの基本的な酸味を形成する。 |
| volatile acidity     | 揮発酸度       | 主に酢酸。多すぎると酢っぽい香りになり、品質を下げる要因。 |
| citric acid          | クエン酸       | 少量含まれる酸。フレッシュさやフルーティーさを与える。 |
| residual sugar       | 残糖分         | 発酵後に残った糖分。赤ワインは低く、甘口ワインでは高い。 |
| chlorides            | 塩化物イオン   | ワイン中の塩分。多いと塩味や不快な味につながる。 |
| free sulfur dioxide  | 遊離亜硫酸     | 抗酸化作用を持つSO₂の一部。ワインの保存性に影響する。 |
| total sulfur dioxide | 総亜硫酸       | ワイン中のSO₂全体量。多すぎると香味を損なう可能性あり。 |
| density              | 密度           | アルコールや糖分の量と関係。アルコールが多いと密度は低くなる傾向。 |
| pH                   | pH値           | 酸性度の指標。低いほど酸味が強く感じられる。 |
| sulphates            | 硫酸塩         | 抗酸化性に寄与。適量は風味や保存性を高める。 |
| alcohol              | アルコール度数 | ワインのボディ感や風味に直結し、品質評価に大きく影響する。 |
| quality              | 品質           | ワインの品質。高いほど良い（ターゲット変数）。 |


### 実験により得た知見
実験では、scikit-learnを用いて多層パーセプトロンで `quality` を予測した場合を実験しました。実験の条件を以下に箇条書きする。

- ホールドアウト法で訓練データを80%, テストデータを20%と分割している。
- 前処理として標準化を導入している。
- 探索したハイパーパラメータは以下の総当たり。`standarization`は標準化のON, OFFを表している。それ以外はscikit-learnのMLPRegressorのドキュメントを参照のこと。
    - `standarization`: `True`, `False`
    - `learning_rate_init`: `0.1`, `0.01`, `0.001`
    - `hidden_layer_sizes`: `[10]`, `[100]`, `[1000, 100]`
    - `activation`: `identity`, `logistic`, `relu`, `tanh`
- 評価指標は以下の3つ
    - RMSE
    - MAE
    - R2

それぞれの評価指標で最も優良な値とハイパーパラメータの組み合わせを以下に示します。
- RMSE: 0.592
    - `standarization`: `True`
    - `learning_rate_init`: `0.01`
    - `hidden_layer_sizes`: `[10]`
    - `activation`: `logistic`
- MAE: 0.454
    - `standarization`: `True`
    - `learning_rate_init`: `0.001`
    - `hidden_layer_sizes`: `[100]`
    - `activation`: `relu`
- R2: 0.467
    - `standarization`: `True`
    - `learning_rate_init`: `0.01`
    - `hidden_layer_sizes`: `[10]`
    - `activation`: `logistic`

以上のようなベースラインを得た。MAEは直感的に、数値からどの程度外れているかを表しており、平均的に±0.454ほどズレているということがわかる。
`quality`が大きな値においてはそれなりに良い値かもしれないが、例えば`3`の場合だと、大きなズレということになる。

ひとまずはこの結果をベースラインとして、今後の予測モデル改善及び分析を行う。
